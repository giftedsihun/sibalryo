# 프로젝트 기술 보고서: 너의 목소리로

## 1. 프로젝트 개요

**프로젝트명**: 너의 목소리로 (Your Voice)  
**장르**: 비주얼 노벨 / 인터랙티브 게임  
**플랫폼**: Windows 데스크탑 (Electron 기반 네이티브 앱)  
**언어**: 한국어  

### 핵심 특징
사용자가 자신의 목소리를 직접 AI에 학습시켜, 게임 내 캐릭터들이 사용자 목소리를 복제(클로닝)하여 대사를 실제 음성으로 출력하는 **AI 음성 복제 비주얼 노벨 게임**이다.

## 4. 게임 구조 및 콘텐츠

### 4.1 등장 캐릭터 (3명)
| 캐릭터 | 설명 | 루트 |
|--------|------|------|
| **하루 (Haru)** | 핑크 머리, 안경, 완벽주의 반장 | 하루 루트 |
| **세진 (Sejin)** | 검은 머리, 이어링, 밴드부 기타리스트 | 세진 루트 |
| **다은 (Daeun)** | 보라색 단발, 도서관 독서광 | 다은 루트 |

### 4.2 스토리 구조
- **프롤로그**: 전학 첫날, 세 캐릭터 소개
- **챕터 1**: 캐릭터 선택 및 첫 만남
- **챕터 2**: 각 캐릭터의 비밀/내면 탐구
- **챕터 3**: 고백 씬
- **엔딩**: 총 **7가지** (하루 A/B, 세진 A/B, 다은 A/B, 시크릿 엔딩)

### 4.3 성격 시스템 (Personality System)
각 캐릭터마다 **슬라이더 3개**로 성격을 커스터마이징 가능하며, 설정값에 따라 대사가 다르게 출력된다.

- **하루**: 따뜻함↔차가움 / 밝음↔조용함 / 장난기↔진지함
- **세진**: 에너지↔차분함 / 말↔침묵 / 반응↔무관심
- **다은**: 수다↔과묵함 / 감성↔이성 / 개방↔폐쇄

### 4.4 호감도 시스템 (Affection System)
- 각 캐릭터당 0~100 범위의 호감도 수치 존재
- 선택지에 따라 호감도 증감
- 세이브 데이터는 `localStorage`(Zustand persist)에 저장

### 4.5 배경 씬 목록 (6종)
`classroom`, `rooftop`, `library`, `musicRoom`, `park`, `title`

## 5. AI 음성 복제 시스템

### 동작 흐름
```
1. [사용자] 게임 시작 전, 자신의 목소리 30초 녹음 or 파일 업로드
      ↓
2. [프론트엔드] POST /clone 엔드포인트로 오디오 파일 전송
      ↓
3. [AI 서버] 참조 오디오 저장 → voice_id (UUID) 반환
      ↓
4. [게임 진행] 캐릭터 대사 출력 시점에 POST /tts 호출
      ↓
5. [AI 서버] OmniVoice 모델로 voice_id 참조 → WAV 스트리밍 반환
      ↓
6. [프론트엔드] Web Audio API로 실시간 재생
```

### API 엔드포인트 (port 8000)
| 엔드포인트 | 메서드 | 설명 |
|------------|--------|------|
| `GET /` | GET | 서버 상태 확인 (health check) |
| `POST /clone` | POST | 참조 음성 등록, voice_id 반환 |
| `POST /tts` | POST | 텍스트 → 복제 음성 WAV 생성 |

### 지원 음성 파일 형식
`audio/mpeg`, `audio/wav`, `audio/mp4`, `audio/ogg`, `audio/x-m4a`, `audio/m4a`

## 6. 배포 패키지 구성

### 최종 Setup.exe 내용물 (압축 전 약 8~9GB)
```
너의 목소리로 Setup 1.0.0.exe
 ├── 게임 코드 (React + Electron)        ~100 MB
 ├── omnivoice/
 │   ├── python/                         ~63 MB  ← 독립형 Python 3.12 런타임
 │   ├── venv/Lib/site-packages/         ~5 GB   ← PyTorch, FastAPI 등 패키지
 │   ├── hf_cache/                       ~3.1 GB ← OmniVoice AI 모델 가중치
 │   └── game_server.py                  ← AI 서버 진입점
```

> **핵심**: 사용자가 Python, CUDA, 패키지를 별도로 설치할 필요 없이 `Setup.exe` 하나로 완전 구동

## 7. 주요 개발 이슈 및 해결

| 이슈 | 원인 | 해결방법 |
|------|------|----------|
| `failed to fetch` | Electron CSP가 `127.0.0.1` 차단 | `index.html`의 `connect-src`에 `http://127.0.0.1:8000` 추가 |
| AI 서버 미시작 | `실행.bat`이 Electron만 실행, AI 서버 미포함 | `bat` 파일에서 Python 서버 선 실행 로직 추가 |
| `localhost` vs `127.0.0.1` 충돌 | 최신 Windows의 IPv6/IPv4 해석 차이 | 모든 엔드포인트를 `127.0.0.1`로 통일 |
| venv Python 이식 불가 | uv가 생성한 venv가 특정 경로의 Python에 의존 | Python 런타임 자체를 패키지에 번들, `PYTHONPATH` 환경변수로 site-packages 주입 |
| AI 모델 타 PC 미존재 | HuggingFace 캐시가 개발 PC에만 존재 | `~/.cache/huggingface/hub/models--k2-fsa--OmniVoice` 전체를 번들에 포함 |
| 심볼릭 링크 압축 실패 | Windows 기본 권한 부족 | 관리자 권한 PowerShell에서 빌드 실행 |

## 9. 실행 환경 요구사항 (최종 배포 기준)

| 항목 | 요구사항 |
|------|----------|
| **OS** | Windows 10/11 (x64) |
| **RAM** | 최소 8GB (VRAM 포함 16GB 권장) |
| **GPU** | NVIDIA GPU (CUDA 지원) 권장, 없으면 CPU로 동작 (느림) |
| **저장공간** | 설치 시 약 8~9GB 필요 |
| **인터넷** | 설치 후 불필요 (완전 오프라인 실행 가능) |
| **별도 설치 필요** | 없음 (Python, CUDA 드라이버 등 모두 내장) |
